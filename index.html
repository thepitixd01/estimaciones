Estimaciones

Estimación Puntual: Usa un valor único para estimar un parámetro desconocido, como el tiempo promedio de carga de una página web.

Estimación de Intervalo de Confianza: Proporciona un rango donde es probable que se encuentre el verdadero valor del parámetro, como el tiempo de compilación de un proyecto.

Estimación Puntual: La estimación puntual se refiere al proceso de usar una única cifra o "punto" como una estimación de un parámetro desconocido en una población. Es decir, se utiliza una muestra para calcular un valor que se considera una buena aproximación del valor verdadero del parámetro en toda la población.

Estimación Puntual: Este es un ejercicio interactivo acerca de la Estimación Puntual. Imagina que eres un desarrollador de una página web y estás tratando de mejorar el rendimiento de la misma. Quieres estimar el tiempo promedio que le toma a los usuarios cargar la página web, sin embargo, no tienes acceso a todos los tiempos de carga posibles, por lo que decides realizar una muestra.
Tu objetivo es estimar el tiempo promedio de carga de la página web para todos los usuarios. Para hacerlo, necesitas tomar una muestra de los tiempos de carga de la página y luego calcular la estimación puntual de tiempos de carga de la página.

Estimación Puntual: Debes ingresar en los campos disponibles el tiempo de carga que le toma a cada persona entrar a la página web. Puede ser cualquier número inventado (entre enteros o decimales) o preguntarle a cinco personas cercanas cuánto tiempo le toma ingresar a esta misma página web y tomarle el tiempo.
Los valores ingresados son una muestra representativa del comportamiento de la página web para un grupo de usuarios. De aquí, vamos a estimar el tiempo promedio de carga para todos los usuarios.

Estimación Puntual: El promedio es una medida estadística comúnmente utilizada para representar un valor central de un conjunto de datos. En este caso, nos interesa obtener el promedio del tiempo de carga. El cálculo del promedio para una muestra de datos se realiza sumando todos los valores y luego dividiendo esa suma entre el número total de datos en la muestra (en este caso se suman todos los valores ingresados y luego se divide entre 5).

Estimación de Intervalos: La estimación de intervalos es un concepto que se usa cuando se quiere aproximar un valor dentro de un rango, en lugar de un valor exacto. Es útil cuando se tiene tiene incertidumbre o cuando los datos no son completamente precisos.

Estimación de Intervalos: Al igual que el ejercicio anterior sobre la Estimación Puntual, este ejercicio será interactivo de igual forma, por lo que tendrá que facilitar los datos que son necesarios para completar la prueba y así generar los resultados. Imagina que estás midiendo el tiempo de compilación de un proyecto en un servidor y quieres estimar el tiempo promedio que toma compilar el proyecto bajo ciertas condiciones de carga. Hay una casilla donde puede ingresar la cantidad de números de mediciones de los tiempos de compilación (en segundos, es necesario ingresar un mínimo de tres números para realizar la operación, además de poder ingresar más de cinco o la cantidad facilitada para completar el ejercicio), la cantidad de números ingresados servirán como muestra para este ejercicio.

Estimación de Intervalos: Vamos a calcular el intervalo de confianza. Hay un total de 7 tiempos de compilación (7, 6, 9, 12, 4, 8, 10). Ingresa los tiempos de compilación uno a uno.

Estimación de Intervalos: Se generará el resultado de este ejercicio una vez ingrese el número de tiempos totales facilitadas, al igual que con cualquier número de tiempo totales que desee aplicar. A continuación, se explicará cómo se llegó a ello: tras obtener la muestra de tiempos de compilación en segundos, se calculó la media y la desviación estándar; para calcular la media se sumaron todos los números ingresados entre sí y luego se dividió entre el número de datos (n) en total; es decir, si se ingresaron los 7 tiempos facilitadas arriba, éstos se sumaron y luego se dividieron entre 7.

Estimación de Intervalos: Se continúa ahora calculando la desviación estándar, donde (xi) son los valores de la muestra, (μ) es la media de la muestra y (n) es el número de datos. Después de recopilar nuestros datos obtenidos hasta ahora, se le restó la media a cada valor de la muestra para así obtener las desviaciones de la media. Por consiguiente, se elevaron al cuadrado los resultados obtenidos de cada uno y luego se sumaron entre sí. La varianza es el promedio de las desviaciones al cuadrado. Como estamos trabajando con una muestra (no con toda la población de compilaciones posibles), debemos dividir entre n - 1 (es decir, 6 en este caso, ya que n = 7 si seguimos con el ejercicio con los datos facilitados desde el principio) para obtener una estimación más precisa. Esto se llama la varianza muestral.

Estimación de Intervalos: Finalmente, la desviación estándar es simplemente la raíz cuadrada de la varianza. Si seguimos el ejercicio planteado, se habría tomado la raíz cuadrada de 7. Una vez se tiene la desviación estándar, se utiliza para calcular el intervalo de confianza de un porcentaje dado (en nuestro caso, un intervalo de confianza del 95%). Este intervalo te da un rango dentro del cual puedes esperar que se encuentren los tiempos de compilación, basándote en la muestra de datos que se ha recogido. Para el intervalo de confianza, multiplicamos la desviación estándar por un valor z (que depende del nivel de confianza que quieras: en este caso, 1.96 para un 95%). Luego, sumamos y restamos este valor a la media para obtener el rango del intervalo. En el ejercicio, si la media es 8 y la desviación estándar es 2.65, y tenemos 7 datos, el intervalo de confianza del 95% sería 1.96. Por lo tanto, el intervalo de confianza sería el resultado que genera el programa. El intervalo de confianza del 95% para los tiempos de compilación es aproximadamente (6.04, 9.96) segundos. Esto significa que, con un 95% de certeza, los tiempos de compilación de futuras muestras estarán entre esos valores.
